# 🌐 Tipos de Aplicações Web

A classificação de aplicações web pode variar dependendo da referência utilizada. Uma forma comum é classificá-las pelo **propósito**.

Principais tipos:

- 🏢 **Institucional**  
  Apresenta informações sobre uma empresa ou organização.

- 📰 **Blog**  
  Focado em publicação de artigos e conteúdos.

- 🛒 **E-commerce**  
  Plataforma para venda de produtos ou serviços.

- 🎯 **Landing Page**  
  Página focada em conversão ou campanhas específicas.

- ⚙️ **Plataforma**  
  Sistemas web com múltiplas funcionalidades (ex: SaaS, redes sociais).

# 📌 Escopo e Backlog do Projeto

O **escopo** define claramente o que será desenvolvido no projeto.

## Importância do escopo

- Facilita comunicação entre equipe e cliente
- Define objetivos claros
- Evita mudanças inesperadas
- Ajuda na organização do desenvolvimento

## Elementos de um escopo bem definido

- Objetivo do sistema
- Funcionalidades principais
- Público-alvo
- Tecnologias utilizadas
- Requisitos de usabilidade

---

# 🧾 Exemplo de Escopo

Projeto: **Site institucional para empresa XYZ**

Objetivos:

- Apresentar identidade da empresa
- Exibir produtos e serviços
- Facilitar contato com clientes

Funcionalidades:

- Homepage
- Página sobre a empresa
- Blog
- Formulário de contato

Tecnologias possíveis:

- HTML
- CSS
- JavaScript
- CMS

Características do sistema:

- 📱 Responsivo
- 🧭 Navegação intuitiva
- ♿ Acessível

# 📋 Planejamento e Gestão do Backlog

O **backlog** organiza todas as tarefas e funcionalidades do projeto.

Objetivos:

- Facilitar comunicação entre equipe e cliente
- Melhorar experiência do usuário
- Organizar desenvolvimento
- Definir prioridades

Uma prática comum é utilizar **User Stories**.

---

# 👤 User Stories

Formato padrão:

Como (quem?)  
Eu quero (o quê?)  
Para (por quê?)

### Exemplo

Como visitante do site  
Eu quero acessar a página "Sobre"  
Para conhecer melhor a empresa e entender sua história e valores.

Benefícios:

- foco no usuário
- requisitos claros
- fácil priorização

# 📊 Gestão de Tasks

Uma forma comum de organizar tarefas é utilizando **Kanban**.

Exemplo de colunas:

- Backlog
- To Do
- In Progress
- Testing
- Done

Ferramentas comuns:

- Trello
- Jira
- Notion
- ClickUp

Vantagens:

- método visual
- organização do fluxo de trabalho
- melhor colaboração

# 🧱 Wireframes

Wireframe é um **esboço da interface do sistema**.

Representa:

- estrutura da página
- organização dos elementos
- fluxo de navegação

## Tipos de Wireframes

### Low Fidelity

- simples
- rápido
- foco na estrutura

### High Fidelity

- detalhado
- próximo do design final

# 🧪 Atividade

1. Definir o tema do projeto do grupo.

2. Pesquisar aplicações web similares:

- analisar pontos fortes
- analisar pontos fracos

3. Descrever:

- arquitetura da aplicação
- fluxo de navegação
- componentes principais

5. Criar **wireframes das telas principais**.

6. Criar um **quadro Kanban no Trello** com as tarefas do projeto.
